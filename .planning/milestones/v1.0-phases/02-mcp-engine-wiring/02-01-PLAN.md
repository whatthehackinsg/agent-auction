All bigint fields use trailing `n` convention: `"48522356...n"` → strip `n`, pass to `BigInt()`.

```json
  "agentSecret": "48522356...n",

  "capabilities": [{ "capabilityId": "1n" }],
  "leafHashes": ["5355868...n"],
  "capabilityMerkleRoot": "14499290...n",
  "registrationCommit": "0x47b3e1..."

```

### Engine nullifier coupling contract (READ-ONLY — DO NOT MODIFY engine code)

When a ZK membership proof is present, the engine:
1. Extracts `nullifier = publicSignals[MEMBERSHIP_SIGNALS.NULLIFIER]` (index 2, string)
2. Converts to `nullifierBigInt = BigInt(nullifier)`
3. Passes `nullifierBigInt` into EIP-712 `verifySignature()` as the `Join.nullifier` field

The MCP signer MUST use the SAME Poseidon nullifier value when signing — otherwise the engine will reject with `Invalid EIP-712 signature`.

### Known project pattern: zeroHashes cast

`buildPoseidonMerkleTree()` returns `{ root, layers }` in its TypeScript type, but the actual object also includes `zeroHashes`. Cast as `(treeResult as any).zeroHashes` — this pattern is established in `packages/crypto/tests/circuits.test.ts`.

### BidRange maxBudget=0 sentinel

When engine returns `maxBid: "0"` (uncapped auction), substitute `BigInt(2 ** 48)` as the `maxBudget` circuit input. The circuit enforces `reservePrice <= bid <= maxBudget`, so `maxBudget=0` would fail.
