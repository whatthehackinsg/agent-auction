**Implementation** (in engine/src/index.ts):

Add a `GET /stats` route handler AFTER the existing `GET /auctions` route (around line 246). The route:

1. Runs three parallel D1 queries via `Promise.all`:
   - Query 1: `SELECT COUNT(*) AS total_auctions, SUM(CASE WHEN status = 1 THEN 1 ELSE 0 END) AS active_auctions, SUM(CASE WHEN status = 3 THEN 1 ELSE 0 END) AS settled_auctions, COALESCE(SUM(CAST(deposit_amount AS INTEGER)), 0) AS total_usdc_bonded FROM auctions`
   - Query 2: `SELECT COUNT(*) AS total_bids FROM events WHERE action_type = 'BID'`
   - Query 3: `SELECT COUNT(DISTINCT agent_id) AS unique_agents FROM events WHERE action_type IN ('JOIN', 'BID')`

2. Returns JSON response:
```typescript
return c.json({
  totalAuctions: totals?.total_auctions ?? 0,
  activeAuctions: totals?.active_auctions ?? 0,
  settledAuctions: totals?.settled_auctions ?? 0,
  totalUsdcBonded: String(totals?.total_usdc_bonded ?? 0),
  totalBids: bids?.total_bids ?? 0,
  uniqueAgents: agents?.unique_agents ?? 0,
}, 200, {
  'Cache-Control': 'public, max-age=10, stale-while-revalidate=30',
})
```

Important notes:
- Use `COALESCE(..., 0)` in SQL for `total_usdc_bonded` to handle NULL from empty SUM
- `totalUsdcBonded` is returned as a string (consistent with USDC amounts elsewhere in the API)
- Filter unique agents to `action_type IN ('JOIN', 'BID')` per research pitfall #2 — restricts to meaningful participation events
- No auth required, no x402 gating, no admin key check — this is a public endpoint
- No new imports needed
