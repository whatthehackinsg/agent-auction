**3. Rewrite StatCard.tsx with shimmer + glow wrapper:**

Wrap `PixelPanel` in a parent `div` that carries the glow class and shimmer overlay. Do NOT modify PixelPanel itself.

Structure:
```tsx
<div className={cn('group relative', `stat-glow-${accent}`)}>
  {/* Shimmer border overlay */}
  <div
    className="animate-stat-shimmer pointer-events-none absolute inset-0 z-20 overflow-hidden"
    aria-hidden="true"
  >
    <div
      className="absolute inset-0"
      style={{
        background: `linear-gradient(90deg, transparent 0%, rgba(${tone.glowRgb}, 0.15) 50%, transparent 100%)`,
        backgroundSize: '200% 100%',
        maskImage: 'linear-gradient(#fff 0 0) content-box, linear-gradient(#fff 0 0)',
        maskComposite: 'exclude',
        WebkitMaskComposite: 'xor',
        padding: '2px',
      }}
    />
  </div>

  <PixelPanel accent={accent} headerLabel={headerLabel} className="relative z-10">
    {/* existing content unchanged */}
  </PixelPanel>
</div>
```

Key implementation notes:
- The glow class (`stat-glow-mint` etc.) is a STATIC class name string using template literal with the `accent` prop directly (e.g., `` `stat-glow-${accent}` ``). Since `accent` is one of 4 known values and all 4 class names appear in globals.css, Tailwind will not purge them.
- The `mask-composite: exclude` + `padding: 2px` technique creates a hollow mask so the shimmer gradient only shows along the border edge, not over content.
- Include `-webkit-mask-composite: xor` for Safari compatibility.
- The shimmer inner div uses `will-change: background-position` is NOT needed (the `animation` property already hints the browser). Do not add it.
- Keep the `displayValue ?? animated.toLocaleString()` logic exactly as-is.

**2. Create AuctionStatsSection.tsx:**

New file at `frontend/src/components/landing/sections/AuctionStatsSection.tsx`.

```tsx
'use client'
import { SectionShell } from '@/components/landing/SectionShell'
import { StatCard } from '@/components/stats/StatCard'
import { UsdcStatCard } from '@/components/landing/sections/PlatformStatsSection'
import { usePlatformStats } from '@/hooks/usePlatformStats'
import { cn } from '@/lib/utils'

export function AuctionStatsSection({ className }: { className?: string }) {
  const { stats, isLoading, error } = usePlatformStats()

  if (error) return null

  return (
    <SectionShell tag="[ :: AUCTION_STATS :: ]" showBraces={false} className={cn('py-8 md:py-12', className)}>
      <div className="grid grid-cols-1 gap-3 sm:grid-cols-3 md:gap-4">
        <StatCard
          accent="mint"
          label="Total Auctions"
          value={isLoading ? 0 : (stats?.totalAuctions ?? 0)}
          headerLabel="auctions.total"
        />
        <StatCard
          accent="rose"
          label="Active Auctions"
          value={isLoading ? 0 : (stats?.activeAuctions ?? 0)}
          headerLabel="auctions.active"
        />
        <UsdcStatCard
          value={stats?.totalUsdcBonded ?? '0'}
          isLoading={isLoading}
        />
      </div>
    </SectionShell>
  )
}
```

Per user decision: 3 stats only (Total Auctions/mint, Active Auctions/rose, USDC Bonded/gold). Layout: `grid-cols-1` on mobile, `sm:grid-cols-3` on desktop (single row of 3).
