<objective>
Add engine-side ERC-721 tokenURI resolution, NftEscrow deposit status reads, and propagate resolved NFT metadata through all engine API responses.

Purpose: The engine is the single source of truth for auction data. By resolving tokenURI at creation time and caching in D1, all downstream consumers (MCP tools, frontend, WebSocket) automatically get rich NFT metadata without separate resolution logic. NftEscrow deposit reads add a trust signal showing whether the NFT is actually in custody.

Output: D1 migration, two new lib modules (nft-metadata.ts, nft-escrow.ts), updated addresses.ts with NftEscrow address, enriched API responses on GET /auctions, GET /auctions/:id, and GET /auctions/:id/manifest.
</objective>
