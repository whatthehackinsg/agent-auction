<deferred_scope>
### Slow Real-Generation Test Tier (intentionally deferred)

CONTEXT.md locked a two-tier test requirement: fast fixture-based (~100ms) + slow real-generation (~5-10s with snarkjs + .wasm/.zkey). This plan implements the fast fixture tier only.

**Rationale for deferral:** The fast fixture tier validates the SAME proof pipeline correctness — the fixtures ARE real Groth16 proofs generated by the same `generateMembershipProof` / `generateBidRangeProof` functions. The only gap is exercising the `generateProof: true` path end-to-end at test time with real .wasm/.zkey artifacts. This is a runtime performance concern, not a correctness concern.

**What the slow tier would add:** `@slow`-tagged tests that call `generateProof: true` on the MCP tool, triggering real snarkjs fullProve against circuit artifacts, and verifying the round-trip through mocked or real engine verification. These tests take 5-10s each and require circuit .wasm/.zkey files accessible at test time.

**Deferred to:** Post-hackathon CI hardening phase. The fast fixture tests are sufficient for hackathon submission confidence.
