  <action>
Create `docs/participation-guide.md` as the canonical Phase 14 source for participation requirements.

The document must be checklist-first and structured so both humans and AI agents can scan it quickly. Include these sections explicitly:

1. A short support matrix labeled `Supported`, `Advanced`, and `Future` covering:
   - `AgentKit + CDP Server Wallet` as the supported target stack
   - the current raw-private-key MCP path as an advanced bridge for power users
   - `Agentic Wallet` as future/not yet protocol-verified
   - Base Sepolia as the only supported network for this standard
   - EIP-4337/account-abstraction context as optional background, not a requirement
2. The active-participant baseline, defined as an active bidder baseline rather than an observer baseline.
3. The wallet capability checklist:
   - one persistent Base Sepolia owner wallet remains the ERC-8004 owner, action signer, and bond/refund wallet
   - the wallet path can sign secp256k1 / EIP-712 auction actions
   - the wallet path can pay Base Sepolia gas and manage USDC bond/refund flows
   - the runtime can preserve or reference compatible ZK state for JOIN/BID
4. The required assets and concise config-input checklist:
   - Base Sepolia ETH for gas
   - Base Sepolia USDC for bond flows
   - minimal config inputs such as engine URL, Base Sepolia RPC, identity target, and ZK state location, without dumping the full `.env.example`
5. Supported entry paths:
   - platform-managed onboarding via `register_identity`
   - externally prepared ERC-8004 identity plus compatible ZK state
6. Fallback policy:
   - if the operator cannot satisfy active participation, route them to read-only observation or the advanced bridge
   - do not imply unsupported stacks are fully active-participant ready
7. Human assistance boundary:
   - explicitly state that acceptable human help is bootstrap-only: initial funding, credential connection, or launching the flow once
   - state that ongoing participation is agent-driven after setup
   - if the runtime still needs repeated human intervention, the operator should use read-only observation or the advanced bridge instead of claiming the supported standard path
8. Phase boundary note:
   - Phase 15 implements the AgentKit-compatible adapter
   - Phase 16 publishes the external skill/playbook
   - this phase only defines the standard and guidance surfaces
