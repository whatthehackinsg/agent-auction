**1. engine/src/handlers/actions.ts — Rewrite the wallet verification block in handleJoin (lines 259-270):**

Remove the cache logic (`walletVerified:${action.agentId}` reads and writes). Replace with direct on-chain verification every time. Change the error handling to throw structured error objects instead of generic Error strings:

- If `resolveAgentWallet` returns null: throw an Error with message formatted as `[AGENT_NOT_REGISTERED] agentId ${action.agentId} is not registered on ERC-8004 (${ERC8004_ADDRESS}). Register via register_identity MCP tool or call register(string) on the contract.`
- If wallet doesn't match: throw an Error with message formatted as `[WALLET_MISMATCH] Wallet ${action.wallet} does not match on-chain owner ${resolvedWallet} for agentId ${action.agentId}. Use the wallet that owns this agentId on ERC-8004.`
- Wrap the entire `verifyAgentWallet` call in try/catch. If the import or call throws a non-verification error (RPC failure): throw an Error with message `[IDENTITY_RPC_FAILURE] Could not verify identity for agentId ${action.agentId}: ${originalError}. Check RPC connectivity. Action rejected (fail-closed).`

Also remove the `poseidonRoot:${action.agentId}` cache logic in the Poseidon root fetch block (lines 273-288). Replace with a direct call to `getAgentPoseidonRoot` every time (no cache read/write to DO storage). Wrap in try/catch — if RPC fails, `agentPoseidonRoot` stays undefined (existing behavior is acceptable since it only affects proof cross-check).

Remove the `walletVerified:${action.agentId}` storage.put call entirely. Remove the `poseidonRoot:${action.agentId}` storage.put call entirely.

**2. engine/src/auction-room.ts — Change verifyWallet default (line 491):**

Change from:
```typescript

```
To:

This makes `ENGINE_VERIFY_WALLET` default to true when unset. Only explicitly setting `'false'` disables it.

Add a guard: if `ENGINE_VERIFY_WALLET` is explicitly `'false'` but `ENGINE_ALLOW_INSECURE_STUBS` is NOT `'true'`, log a warning and force `verifyWallet: true`. This ensures the bypass only works in dev mode. Implementation:
```typescript
let verifyWallet = this.env.ENGINE_VERIFY_WALLET !== 'false'
if (!verifyWallet && this.env.ENGINE_ALLOW_INSECURE_STUBS !== 'true') {
```

**3. engine/src/lib/identity.ts — Make verifyAgentWallet distinguish unregistered vs mismatch:**

Update `verifyAgentWallet` return type to include a `reason` field:
```typescript
Promise<{ verified: boolean; resolvedWallet: string | null; reason?: 'not_registered' | 'mismatch' | 'verified' }>
```
- If `resolvedWallet` is null: return `{ verified: false, resolvedWallet: null, reason: 'not_registered' }`
- If wallet doesn't match: return `{ verified: false, resolvedWallet, reason: 'mismatch' }`
- If match: return `{ verified: true, resolvedWallet, reason: 'verified' }`

Do NOT catch RPC errors in `resolveAgentWallet` for the zero-address case — that stays. But if the `identityRegistry.read.ownerOf` call itself throws (network error), let it propagate (remove the catch-all that returns null). Instead, only catch the specific ERC-721 revert for non-existent tokens. Use a targeted approach:
```typescript
} catch (err) {

  const msg = err instanceof Error ? err.message : String(err)
  if (msg.includes('ERC721') || msg.includes('nonexistent') || msg.includes('invalid token')) {
    return null
  }
  // RPC/network errors propagate for fail-closed behavior
  throw err
}
```
