<objective>
Lay the foundation for the supported AgentKit/CDP write path by replacing the MCP server's raw-private-key assumption with a shared wallet-backend abstraction and explicit config contract.

Purpose: the current MCP runtime hardwires `AGENT_PRIVATE_KEY` into config, target resolution, and health reporting. Phase 15 needs a supported wallet backend that can coexist with the advanced raw-key bridge without introducing a second server or a silent fallback.
Output: shared backend/config primitives plus regression coverage that make AgentKit/CDP the supported default path without breaking the advanced bridge or read-only tools.
</objective>
